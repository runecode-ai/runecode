# Product Mission

## Problem

Agentic coding/automation is powerful but unsafe by default: a single compromised or misaligned component can gain broad access (web + repo + shell + secrets) with little auditability, enforceable policy, or clear blast-radius control.

## Target Users

Anyone running agentic software engineering workflows who needs security from day one: individual developers, teams, and organizations that require least-privilege defaults, explicit approvals, and auditable evidence.

## Solution

RuneCode is a QubesOS-inspired, security-first agentic automation platform that treats isolation and cryptographic provenance as co-equal, first-class pillars. Work runs in tightly scoped "isolates" with deny-by-default capabilities, explicit artifact-based data movement, and a tamper-evident audit trail for every action, decision, and output.
