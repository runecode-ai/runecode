# Product Mission

## Problem

Agentic coding and automation are powerful but unsafe by default: a single compromised or misaligned component can gain broad access to the web, repository, shell, and secrets with little auditability, enforceable policy, or clear blast-radius control.

## Target Users

Anyone running agentic software engineering workflows who needs security from day one: individual developers, teams, and organizations that require least-privilege defaults, explicit approvals, and auditable evidence.

## Solution

RuneCode is a security-first agentic automation platform inspired by Qubes OS. It treats isolation and cryptographic provenance as co-equal first-class pillars, so work runs in tightly scoped isolates with deny-by-default capabilities, explicit artifact-based data movement, and a tamper-evident audit trail for every action, decision, and output.
